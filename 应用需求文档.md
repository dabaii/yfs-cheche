一番赏“纯开车模式”H5应用 需求与设计文档

1. 项目概述

本应用是一款专注于一番赏“组队包箱”（开车模式）的二次元风格H5应用。用户通过购买特定数量的席位共同出资买断整箱赏品，满员后系统自动开奖并分配赏品。

1.1 核心设计风格

视觉风格：新旧主义（Neubrutalism）二次元动漫风格。

视觉元素：粗黑边框（4px）、硬阴影、高饱和度配色（明黄、靛蓝、玫瑰粉）、贴纸化标签（Sticker）。

交互反馈：按钮压下动效、次元传送门式弹窗。

2. 角色功能描述

2.1 用户类型

访客 (Guest)：未登录状态，可浏览基本内容，但关键数据（余额、记录）脱敏显示，操作时强制拦截登录。

普通参与者 (User)：已登录，可进行占位购买、充值、地址管理、申请发货、查看战报。

管理员 (Admin)：拥有普通用户所有功能，额外拥有“工厂”入口，可部署新车队、管理据点（门店）。

2.2 核心模块功能

A. 首页大厅 (Quest Board)

冒险布告栏：展示所有进行中和已完结的车队。

车队卡片：

展示任务名称、集结描述、集结进度条。

赏品预览：横向滚动展示本箱所有奖项（A/B/C/F赏）及对应数量，点击可查看大图。

加入队伍：访客弹出登录，用户选择购买席位数（1-N），展示总金币消耗。

狩猎成功：已完结车队文案变更，点击查看双列布局战报（公示每个席位的中奖人及奖品）。

B. 行囊/宝库 (Warehouse)

按车队分类：奖品按照所属车队名称归类展示。

多选机制：点击奖品卡片可进行多选勾选（Checkbox效果）。

申请传送：勾选后右上角出现按钮，跳转至“确认传送”页面。

传送确认页：

支持“快递”与“自提”切换。

关联用户默认地址或后台配置的据点。

C. 勇者中心 (Profile)

数据展示：金币余额、稀有掉落总数、等级/称号。

坐标管理 (地址簿)：

列表展示所有坐标。

新增坐标表单：姓名、手机、详细地址、设为默认勾选。

金币补给：模拟充值功能。

D. 建造工坊 (Admin Factory)

部署新物语 (发车)：

表单填写：任务名称、席位数、单价。

奖项动态配置：支持动态增删奖项行，设置级别、名称、数量。

合法性校验：奖项总数必须等于席位数方可部署。

据点管理 (门店)：管理线下自提点的名称、地址、营业时间。

3. 业务流程图

3.1 用户参与流程 (Mermaid Logic)

graph TD
    A[进入大厅] --> B{是否登录?}
    B -- 否 --> C[浏览脱敏数据]
    C --> D[点击加入/操作]
    D --> E[弹出登录窗]
    E --> F[选择身份登录]
    B -- 是 --> G[查看实时进度]
    G --> H[输入占位数量]
    H --> I{余额足够?}
    I -- 否 --> J[跳转充值]
    I -- 是 --> K[扣款/占位成功]
    K --> L{席位已满?}
    L -- 是 --> M[系统自动洗牌开奖]
    M --> N[生成狩猎战报]
    N --> O[奖品进入行囊]

4. 技术栈建议

前端框架：React 18 + Tailwind CSS (用于样式快速部署)。

状态管理：目前为内部 State，建议后续接入持久化方案。

图标库：Lucide React。

动效库：Tailwind-animate。

5. 保存与云端发布指南

5.1 本地保存

您可以将当前代码保存为 index.html。由于它是单文件 React 架构，您需要确保 HTML 中包含了对应的 CDN 脚本（React, Tailwind, Lucide）。

5.2 免费云端发布 (推荐 Vercel 或 Netlify)

这是目前最快且完全免费的方式：

方案一：Vercel (最简单)

准备环境：在本地创建一个文件夹，新建 index.html（将您的代码放入 React 挂载模板中）或使用标准的 Vite React 项目模板。

托管到 GitHub：创建一个 GitHub 仓库，将项目上传。

连接 Vercel：

登录 Vercel.com。

点击 "Add New" -> "Project"。

导入您的 GitHub 仓库。

点击 "Deploy"。

结果：您将获得一个类似 my-kuji-app.vercel.app 的公网访问地址。

方案二：GitHub Pages (完全静态)

在您的 GitHub 仓库设置 (Settings) 中找到 "Pages"。

选择 "Deploy from a branch"，分支选择 main。

保存后，GitHub 会自动为您生成一个 https://<用户名>.github.io/<项目名>/ 的链接。

方案三：CodeSandbox (即时演示)

打开 CodeSandbox.io。

选择 "React" 模板。

直接将整个 index.jsx 的代码覆盖进去。

右侧预览框的 URL 即可分享给他人查看。

提示：由于当前代码使用了内存模拟数据（State），刷新浏览器数据会重置。若要真正运营，下一步建议增加简单的后端接口（如使用 Supabase 或 Firebase）来存储车队和用户信息。
