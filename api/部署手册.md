# 一番开车 - 后端服务部署手册

本手册涵盖了本地开发环境与上线时所需的基础配置、数据库启动参数、以及如何访问 Django CMS 后台等核心指引。

## 1. 运行环境

本后端系统构建于 `e:\work\yfs-cheche\api` 目录下，所需的前置环境为：
- **Python**: 3.10+ (本项目已在 `api/venv` 下创建虚拟环境并隔离依赖)
- **Docker**: 用于提供独立解耦的数据库服务

## 2. 启动基础数据服务 (MySQL)

为了避免和宿主机的默认 3306 端口发生冲突，本项目使用了 33066 端口进行端口映射转发。

请在终端或 PowerShell 中执行以下命令以创建或唤醒容器：

```powershell
# 启动 MySQL 5.7 独立容器，对外暴露 33066 端口
docker run -d --name yfs-cheche-db -p 33066:3306 -e MYSQL_ROOT_PASSWORD=root -e MYSQL_DATABASE=cheche mysql:5.7
```

**数据库连线信息：**
- **地址 (Host)**: `127.0.0.1`
- **端口 (Port)**: `33066`
- **默认账号 (User)**: `root`
- **默认密码 (Password)**: `root`
- **数据库名 (Database)**: `cheche` (字符集已校准为 `utf8mb4` 防止中日文名称乱码)

> 提示：线上生产环境中请务必修改上述弱密码，并在 `server/settings.py` 的 `DATABASES` 节点中同步更新秘钥配对。

## 3. 运行服务端引擎

在确保数据库容器正在运行的前提下，我们需要激活内置的 Python 运行环境以承载后端代码：

```powershell
# 1. 进到纯后端目录层
cd api

# 2. 激活虚拟环境 (Windows PowerShell)
.\venv\Scripts\Activate.ps1
# 提示: 如果是 Windows CMD 则是 .\venv\Scripts\activate.bat 

# 3. 启动开发服务器 (可指定网关与监听端口)
python manage.py runserver 0.0.0.0:8000
```

> **补充：数据重置与数据迁移备忘 (当换机器部署时)**
> 如果需要在全新的机器上部署，或者清空了所有数据库表，请依次运行以下两个指令重建大纲：
> 1. `python manage.py migrate` (合并执行数据库建表同步)
> 2. `python manage.py createsuperuser` (创建属于你的最高管理权限账号)
> *(注：本次生成时已为你执行过这些操作，只需直接 `runserver` 起运服务器即可。)*

## 4. CMS 与 API 访问指南

当在终端看到类似于 `Starting development server at http://0.0.0.0:8000/` 的提示时，即代表服务器点火成功：

- **👑 CMS 厂长指挥调度系统**
  - **默认访问入口**: [http://127.0.0.1:8000/admin](http://127.0.0.1:8000/admin)
  - **内化超级管理员**: `admin`
  - **超管默认密码**: `admin`
  - *说明*：在该后台内集成了开箱盲盒配置(车队)、勇者(玩家钱包)管理、物流申请发货打单、物品掉落流水等全套管理体系。建议前台应用开启前先在此注入些初始数据。

- **🔌 RESTful 前端网关 API**
  - **请求根地址**: `http://127.0.0.1:8000/api/v1/`
  - *包含了如获取车队伍列 `/api/v1/cars/active`、发起登录 `/api/v1/auth/login` 等资源。接口层标准请求结构及所需参数，请查见前端根目录下的 `接口.md` 文档。*
