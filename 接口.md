# 一番开车项目 API 接口文档 (Restful 规范)

本文档列出了当前前端项目中被抽离成独立调用的模拟 API 操作。所有的接口目前已经统一遵循下述提到的 **Restful 标准包装格式**。方便后续真实后端无缝替换。

## 通用约定

- **接口返回标准结构**
  ```json
  {
      "success": true,   // 业务执行是否成功
      "code": 200,       // Http或业务状态码 (200=成功)
      "data": {},        // 数据载荷 (可以是对象或数组结构)
      "message": "success" // 错误消息或其他提示说明
  }
  ```
- **登录状态校验**
  通常在 HTTP Header 附加 `Authorization: Bearer <Token>` 提交身份凭证进行权限切面拦截。列表中标明 `是否需要登录` 的接口对应此拦截鉴定过程。

---

## 1. 基础数据拉取接口 (启动/刷新阶段加载)

### 1.1 获取用户信息与背包
- **接口地址**: `/api/user/profile`
- **请求方法**: `GET`
- **是否需要登录**: **是**
- **请求参数**: 无
- **返回值 (`data`字段内容)**:
  ```json
  {
      "success": true,
      "code": 200,
      "message": "success",
      "data": {
          "id": "u1",
          "name": "40岁勇者",
          "balance": 5000,
          "items": [...],      
          "isLoggedIn": true,
          "role": "user"       
      }
  }
  ```

### 1.2 获取大厅所有行进中的车队
- **接口地址**: `/api/cars/active`
- **请求方法**: `GET`
- **是否需要登录**: 否
- **请求参数**: 无
- **返回值 (`data`字段内容)**: 包含车队对象的数组。
  ```json
  {
      "success": true,
      "code": 200,
      "message": "success",
      "data": [
          {
              "id": "car-201",
              "name": "海贼王 - 意志的继承",
              "totalSeats": 10,
              "price": 68,
              "status": "open",
              "results": [...]
          }
      ]
  }
  ```

### 1.3 获取当前用户收货坐标
- **接口地址**: `/api/user/addresses`
- **请求方法**: `GET`
- **是否需要登录**: **是**
- **返回值 (`data`字段内容)**: 地址数组
  ```json
  {
      "success": true,
      "data": [
          { "id": 1, "name": "勇者基地", "phone": "138...", "detail": "...", "isDefault": true }
      ]
  }
  ```

### 1.4 获取线下合作门店实体据点
- **接口地址**: `/api/stores`
- **请求方法**: `GET`
- **是否需要登录**: 否
- **返回值 (`data`字段内容)**: 门店数组

### 1.5 获取当前用户发货与物流记录
- **接口地址**: `/api/user/shipping-history`
- **请求方法**: `GET`
- **是否需要登录**: **是**
- **返回值 (`data`字段内容)**: 发货历史对象数组

## 2. 用户交互与操作提交接口 (修改数据表盘)

### 2.1 用户/管理员登录请求
- **接口地址**: `/api/auth/login`
- **请求方法**: `POST`
- **请求参数**: `{ "role": "user" }`
- **返回值**: 
  ```json
  { "success": true, "code": 200, "data": { /* User对象 */ }, "message": "success" }
  ```

### 2.2 登出 / 注销当前次元
- **接口地址**: `/api/auth/logout`
- **请求方法**: `POST`
- **返回值**: `{ "success": true, "code": 200, "data": null, "message": "success" }`

### 2.3 添加/记录新坐标 (创建地址)
- **接口地址**: `/api/user/address`
- **请求方法**: `POST`
- **请求参数**: `{ "name": "...", "phone": "...", "detail": "...", "isDefault": false }`
- **返回值**: 返回带有新ID的完整地址放入`data`中。

### 2.4 篡改现存的坐标描述 (更新地址)
- **接口地址**: `/api/user/address/:id`
- **请求方法**: `PUT`
- **返回值**: `{ "success": true, "code": 200, "data": null, "message": "success" }`

### 2.5 废除遗失标点 (删除地址)
- **接口地址**: `/api/user/address/:id`
- **请求方法**: `DELETE`
- **返回值**: `{ "success": true, "code": 200, "data": null, "message": "success" }`

### 2.6 加入集结车队 / 占购席位
- **接口地址**: `/api/cars/:id/join`
- **请求方法**: `POST`
- **请求参数**: `{ "count": 5, "totalCost": 340.00 }`
- **返回值**: `{ "success": true, "code": 200, "data": null, "message": "success" }`

### 2.7 申请行囊物资发货
- **接口地址**: `/api/user/shipping`
- **请求方法**: `POST`
- **请求参数**: `{ "items": [], "method": "express" }`
- **返回值**: `{ "success": true, "code": 200, "data": { /* 物流记录对象 */ }, "message": "success" }`

### 2.8 魔法充值金币
- **接口地址**: `/api/user/balance/recharge`
- **请求方法**: `POST`
- **请求参数**: `{ "amount": 1000 }`
- **返回值**: `{ "success": true, "code": 200, "data": null, "message": "success" }`

### 2.9 (仅限厂长) 部署全新物语关卡车队
- **接口地址**: `/api/admin/cars`
- **请求方法**: `POST`
- **请求参数**: 包含车队表单所有详情。
- **返回值**: 成功在 `data` 存放生成的后台车队实例包。
